# Product analytics portfolio: funnel, A/B test and retention

Учебный портфолио-проект для позиции **Junior Data Analyst**.  
Проект показывает полный цикл аналитической задачи: генерация и проверка данных, SQL-запросы, расчет продуктовых метрик, A/B-тест, когортный анализ, визуализация и выводы для бизнеса.

Данные синтетические и создаются скриптом `scripts/generate_data.py`. Это сделано специально, чтобы проект можно было свободно публиковать на GitHub и воспроизводить без закрытых датасетов.

## Задача

Представим, что продуктовая команда тестирует новый сценарий оплаты в приложении. Нужно понять:

- как пользователи проходят воронку от визита до покупки;
- есть ли прирост конверсии и ARPU в тестовой группе;
- какие каналы и платформы дают больше выручки;
- как выглядит недельное удержание по когортам;
- можно ли рекомендовать тестовый вариант к дальнейшему rollout.

## Что сделано

- Проведена проверка данных и расчет базовых метрик.
- Через SQL рассчитана воронка по группам A/B-теста.

- Проведен статистический тест для сравнения групп.

## Ключевые метрики

- **Conversion rate** - доля пользователей, совершивших покупку.
- **ARPU** - средняя выручка на пользователя.
- **ARPPU** - средняя выручка на платящего пользователя.
- **Retention** - доля активных пользователей по неделям после регистрации.
- **Relative uplift** - относительное изменение метрики в тестовой группе относительно контрольной.

## Как запустить

```bash
python -m venv .venv
.venv/Scripts/activate
pip install -r requirements.txt
python scripts/generate_data.py
python scripts/run_analysis.py
```

После запуска результаты появятся в папке `outputs`.

## Выводы

Тестовая группа показывает более высокую конверсию в покупку и ARPU. Перед полным rollout стоит дополнительно проверить удержание и поддержку пользователей, затем масштабировать вариант на наиболее сильные каналы и платформы.
